## Responsive Design Testing (Responsively App)

To test and demonstrate the responsive behaviour of the application across multiple devices and screen sizes, the **Responsively App** was used.

Responsively is a cross-platform desktop application that allows developers to preview a website simultaneously across a range of predefined devices, including mobile phones, tablets, and desktop viewports. It was selected as a reliable alternative to web-based tools such as *Am I Responsive*, which can have issues loading deployed Heroku applications.

### Installation
The Responsively App was downloaded from the official website and installed locally on the development machine. Once installed, the application allows any deployed or local URL to be loaded and viewed across multiple device frames at the same time.

### Usage
The deployed Heroku URL for the project was loaded into Responsively. The application automatically rendered the site across several device profiles, including:

- iPhone 12 Pro (mobile)
- iPad (tablet)
- MacBook Pro (desktop)

This made it possible to verify that layout, navigation, typography, and interactive components (such as menus, forms, carts, and admin interfaces) adapted correctly at different screen widths.

Responsivelyâ€™s built-in screenshot functionality was then used to capture consistent, side-by-side images for each device. These screenshots were saved and included in the project documentation as evidence of responsive design testing across user-facing and admin views.

### Outcome
Using Responsively helped confirm that the application remains fully functional and visually consistent across mobile, tablet, and desktop devices, supporting a mobile-first and accessibility-aware design approach.
